Column Selection in Pandas - and Pyjanitor¶

This article highlights various ways to select columns using Pandas and Pyjanitor.

Pyjanitor is an open-source project that extends Pandas chaining methods with a verb-based API.

The examples used here are based off the excellent article by Susan Baert.

The data file can be accessed here

Selecting Columns¶

You can also select single columns with Pyjanitor’s select_columns; a dataframe is always returned:

df.select_columns('genus').head()
genus
0 Acinonyx
1 Aotus
2 Aplodontia
3 Blarina
4 Bos

Note: select_columns only works with labels, and can accept a string, a regular expression, a slice, a function, or a list containing any of the previous types. If a function is provided, the function should be applicable to every seriesin the dataframe.

columns_to_select = ["name", "genus", "sleep_total", "awake"]

Selecting Columns based on Partial Names¶

Pandas has a filter function that makes column selection with partial matching easy:

df.filter(like = 'sleep').head()

Of course, you can select with Pandas loc or Pyjanitor’s select_columns, in combination with Pandas’ string methods:

# this returns a list of booleans
columns_to_select = df.columns.str.contains('sleep')

df.loc[:, columns_to_select].head()

With Pyjanitor’s select_columns, you can pass a function that targets each individual Series in the dataframe:

# this searches through each Series' name for matches
columns_to_select = lambda df: 'sleep' in df.name

df.select_columns(columns_to_select).head()

1 17.0 1.8 NaN
2 14.4 2.4 NaN
3 14.9 2.3 0.133333
4 4.0 0.7 0.666667

Selecting columns by logical expressions¶

The ideas expressed in the previous sections can be combined for some nifty column selection.

Say we wish to select columns that are numeric, and have a mean greater than 10:

# first select only numeric columns
# then select columns where the mean is greater than 10
(df
 .select_dtypes('number')
 .loc[:, lambda df: df.mean()>10]
 .head()
 )
sleep_total awake bodywt
0 12.1 11.9 50.000
1 17.0 7.0 0.480
2 14.4 9.6 1.350
3 14.9 9.1 0.019
4 4.0 20.0 600.000

Alternatively, we can just get the column names and pass them directly to loc

columns_to_select = df.mean().loc[lambda s: s > 10].index

1 NaN 17.0 Owl monkey

To move some columns to the front, you could use Pyjanitor’s reorder function:

df.reorder_columns(['conservation', 'sleep_total']).head()

2 nt 14.4 Mountain beaver Aplodontia herbi Rodentia 2.4 NaN 9.6 NaN 1.350

Or, you could write a function and use within Pandas:

def move_to_the_front(frame, front_columns):
    column_names = list(frame.columns)
    for name in front_columns:
        column_names.remove(name)
    front_columns.extend(column_names)
    return front_columns

df.loc[:, move_to_the_front(df, ['conservation', 'sleep_total'])].head()

Renaming Columns¶

Columns can be renamed with Pandas’ rename method and a dictionary, where you map the old column name to the new column name:

(df
 .loc[:, ['name', 'sleep_total', 'conservation']]
 .rename(columns = {"name": "animal", "conservation": "extinction_threat"})
 .head()
 )
animal sleep_total extinction_threat
0 Cheetah 12.1 lc
1 Owl monkey 17.0 NaN
2 Mountain beaver 14.4 nt
3 Greater short-tailed shrew 14.9 lc
4 Cow 4.0 domesticated

Pyjanitor has a rename_columns function, which is just syntactic sugar for Pandas’ rename:

(df
 .select_columns(['name', 'sleep_total', 'conservation'])
 .rename_columns({"name": "animal", "conservation": "extinction_threat"})
 .head()
 )
animal sleep_total extinction_threat
0 Cheetah 12.1 lc
1 Owl monkey 17.0 NaN
2 Mountain beaver 14.4 nt
3 Greater short-tailed shrew 14.9 lc
4 Cow 4.0 domesticated
